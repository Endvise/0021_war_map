# PRD 문서

## SLG 전장 타워 도달 시간 시뮬레이터

---

## 1. 제품 개요

### 제품명

**SLG 전장 타워 시뮬레이터**

### 목적

둠스데이 라스트 서바이버 같은 SLG 게임의 전장 맵에서, 사용자가 직접 입력한 맵 좌표·타워 비용·건설 시간·시멘트 보유량·분당 생산량을 기반으로 **우리와 상대가 목표 지점까지 몇 시간 뒤 도달 가능한지** 계산하는 웹 기반 시뮬레이터를 만든다.

### 핵심 목표

사용자는 전장 시작 전 또는 진행 중에 다음 내용을 미리 계산할 수 있어야 한다.

```text
우리 연맹이 목표 지점까지 몇 시간 걸리는가?
상대 연맹은 몇 시간 걸리는가?
누가 먼저 목표에 도달하는가?
어느 좌표에서 충돌하는가?
목표까지 시멘트가 충분한가?
몇 번째 타워부터 시멘트 부족으로 대기가 발생하는가?
타워가 전소되면 라인이 끊기는가?
```

---

## 2. 문제 정의

현재 SLG 전장에서는 타워를 이어서 영토를 확장해야 한다.
하지만 실제 전투 중에는 다음 정보를 빠르게 계산하기 어렵다.

```text
목표까지 필요한 타워 수
누적 타워 비용
건설 완료까지 걸리는 시간
상대의 도달 예상 시간
시멘트 부족 발생 시점
전소 중인 타워가 무너지기까지 남은 시간
타워 라인이 끊기는 위험 지점
```

따라서 사용자가 직접 입력한 값을 기반으로 전장 상황을 사전에 시뮬레이션하는 도구가 필요하다.


## 6.3 타워 설정 기능

### 설명

타워의 기본 규칙을 사용자가 직접 설정한다.

### 입력값

```text
타워 영역 반경
타워 간 최대 건설 거리
타워 기본 건설 시간
타워 기본 내구도
전소 시 분당 피해량
건설 가능 조건
타워 비용 증가 방식
```

### 타워 비용 증가 방식

#### 1. 고정 비용

```text
모든 타워 비용이 동일
```

#### 2. 선형 증가

```text
다음 타워 비용 = 기본 비용 + 현재 타워 수 × 증가값
```

#### 3. 비율 증가

```text
다음 타워 비용 = 기본 비용 × 증가율 ^ 현재 타워 수
```

#### 4. 직접 입력표

```text
타워 번호별 비용을 사용자가 직접 입력
```

예시:

```text
28번 타워: 180,000
29번 타워: 190,000
30번 타워: 200,000
31번 타워: 215,000
```

## 6.4 우리/상대 자원 입력 기능

### 설명

### 우리 입력값

```text
현재 보유 시멘트
분당 시멘트 생산량
현재 타워 개수
현재 타워 라인 끝 좌표
타워 건설 시간
건설 속도 보너스
```

### 상대 입력값

```text
추정 보유 시멘트
추정 분당 시멘트 생산량

상대 타워 라인 끝 좌표
상대 타워 건설 시간
상대 건설 속도 보너스
```

### 계산 반영

각 타워 건설 시 다음을 계산한다.

```text
현재 시멘트가 충분한가?
부족하면 몇 분 대기해야 하는가?
건설 중 생산되는 시멘트는 얼마인가?
건설 완료 후 남은 시멘트는 얼마인가?
다음 타워는 언제 시작 가능한가?
```

## 6.5 도달 시간 계산 기능

### 설명

시작점에서 목표 지점까지 타워를 이어서 건설할 경우 필요한 타워 수와 총 시간을 계산한다.

### 기본 계산식

```text
목표까지 거리 ÷ 타워 1개당 전진 가능 거리 = 필요 타워 수
```

```text
총 도달 시간 = 대기 시간 + 타워 건설 시간의 합
```

### 출력값

```text
필요 타워 수
총 건설 시간
시멘트 대기 시간
총 소모 시멘트
도달 예상 시간
도달 시 남은 시멘트
```

### 예시 결과

```text
목표: 중앙 마을 x:2000 y:2000

우리:
필요 타워 수: 8개
도달 시간: 10시간 40분
총 시멘트 소모: 1,850,000
도달 시 잔여 시멘트: 230,000

상대:
필요 타워 수: 7개
도달 시간: 9시간 20분
총 시멘트 소모: 1,620,000
도달 시 잔여 시멘트: 110,000

판정:
상대가 1시간 20분 빠름
```

## 6.6 충돌 지점 계산 기능

### 설명

우리와 상대가 서로 목표 방향으로 타워를 확장할 때, 어느 좌표에서 영역이 맞닿는지 계산한다.

### 계산 조건

```text
우리 시작 좌표
상대 시작 좌표
각자의 타워 전진 거리
각자의 건설 시간
각자의 시멘트 보유량
각자의 분당 생산량
```

### 출력값

```text
예상 충돌 좌표
우리 도달 시각
상대 도달 시각
먼저 도달한 쪽
시간 차이
```

### 예시

```text
예상 충돌 지점: x 1840, y 2110
우리 도달: 8시간 30분 후
상대 도달: 7시간 50분 후
판정: 상대가 40분 빠름
```

## 6.7 타워 전소 시간 계산 기능

### 설명

전소 중인 타워가 언제 파괴되는지 계산한다.

### 입력값

```text
현재 내구도
최대 내구도
분당 연소 피해
분당 수리량
남은 연소 시간
```

### 계산식

```text
실제 분당 감소량 = 분당 연소 피해 - 분당 수리량
```

```text
전소까지 남은 시간 = 현재 내구도 ÷ 실제 분당 감소량
```

### 출력값

```text
전소까지 남은 시간
예상 파괴 시각
라인 끊김 여부
끊길 경우 영향받는 후방 타워 목록
```

---

## 6.8 라인 끊김 시뮬레이션

### 설명

특정 타워가 전소되거나 제거되었을 때, 뒤쪽 타워들이 본진과 연결이 끊기는지 계산한다.

### 요구사항

```text
타워 간 연결 관계 계산
본진과 연결된 타워 목록 표시
연결이 끊긴 타워 목록 표시
위험 타워 강조 표시
```

### 출력 예시

```text
tower_12 전소 시:
연결 유지 타워: 1~12번
연결 끊김 타워: 13~18번
위험 구간: x 1780~1920, y 2100~2180
```

## 6.9 경로 설정 기능

### 설명

목표까지 직선으로 계산하거나, 사용자가 중간 경유지를 설정할 수 있다.

### 경로 모드

```text
직선 경로
경유지 경로

```

### MVP에서는

초기 버전은 아래 2개만 지원한다.

```text
직선 경로
수동 경유지 경로
```

장애물 자동 우회는 후순위 기능으로 둔다.

# 12. 최종 요약

이 프로젝트는 **둠스데이 라스트 서바이버 전장용 사용자 입력 기반 타워 시뮬레이터**다.

핵심은 게임 데이터를 자동으로 가져오는 것이 아니라, 사용자가 인게임에서 확인한 값을 입력하면 아래를 계산해주는 것이다.

```text
목표까지 필요한 타워 수
총 건설 시간
총 시멘트 소모량
시멘트 부족 발생 시점
우리와 상대의 도달 시간 차이
예상 충돌 지점
타워 전소 및 라인 끊김 위험
```

초기 버전은 **계산 정확도**를 우선하고, 이후에 맵 UI와 전소/라인 끊김 같은 고급 기능을 붙이는 방향이 적합하다.
