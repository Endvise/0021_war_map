## 1. 제품 개요

### 제품명
**SLG 전장 타워 시뮬레이터** (War Map Tower Simulator)

### 한 줄 정의
개발자가 미리 세팅한 맵 템플릿을 불러와, 전장마다 바뀌는 값(시멘트·상대 추정치)만 입력하면 도달 시간과 결과를 즉시 계산해주는 웹앱.

| # | 질문 |
|---|------|
| 1 | 우리가 목표까지 몇 시간 걸리는가? |
| 2 | 상대는 몇 시간? 누가 먼저 도달하는가? |
| 3 | 어느 좌표에서 충돌하는가? |
| 4 | 시멘트가 몇 번째 타워부터 부족해지는가? |
| 5 | 전소 중인 타워가 언제 무너지는가? |
| 6 | 현재 점령 시나리오에서 각 연맹의 예상 점수는? |
| 7 | 동맹 관계를 고려한 우리 진영의 총 점수는? |

### 5.3 계산 엔진

**도달 시간**
```
타워 영역: 9×9 (중심 ±4)
연결 조건: 두 타워 영역 사이 간격 ≤ 3칸

타워 1 오른쪽 끝: center_x + 4
타워 2 왼쪽 끝:   center_x2 - 4
간격 = (center_x2 - 4) - (center_x + 4) ≤ 3
→ center_x2 - center_x ≤ 11

최대 전진 거리 = 11 좌표 단위 / 타워 1개

필요 타워 수 = ceil(목표까지 거리 ÷ 11)
총 시간 = Σ(건설 시간 + 시멘트 대기 시간)
시멘트 대기 = max(0, 부족량 ÷ 분당 생산량)

타워 비용 조회 (룩업테이블, 현재 타워 개수 기준):
  탑 1~5   → 10,500
  탑 6~10  → 17,500
  탑 11~15 → 21,000
  탑 16~25 → 24,500
  탑 26~30 → 28,000
  탑 31~35 → 30,700
  탑 36~40 → 36,400
  탑 41~50 → 42,000
```

**충돌 지점**
```
양측이 같은 목표로 확장할 때
각 타워 건설 완료 시각 기준으로 영역 겹침 좌표 계산
```

**전소 시간**
```
실제 분당 감소 = 분당 피해 - 분당 수리
전소까지 시간 = 현재 내구도 ÷ 실제 분당 감소
```

## 5-B. 도시 점령 점수 시스템

### 점수 체계

| 도시 등급 | 점령 점수 | 수량 | 최대 가용 점수 |
|-----------|----------|------|--------------|
| 6도시 | 100,000점 | 8개 | 800,000점 |
| 7도시 | 300,000점 | 4개 | 1,200,000점 |
| 8도시 | 600,000점 | 1개 | 600,000점 |
| **합계** | | **13개** | **2,600,000점** |

### 점수 계산 규칙

```
연맹 점수 = Σ(점령한 도시의 점수)
진영 점수 = 우리 연맹 점수 + 동맹 연맹 점수의 합
```

### 코인추격 (프리즘코인 쟁탈)

7도시에는 도시 점령 점수와 별개로 **프리즘코인 풀**이 존재한다.

| 항목 | 값 | 비고 |
|------|-----|------|
| 7도시 초기 프리즘코인 | 450,000점 | 도시당 고정 |
| 코인추격 1회 이동량 | 90,000점 | 전쟁 선포 → 성공 시 이동 |
| 7도시당 최대 추격 가능 횟수 | 5회 | 450,000 ÷ 90,000 |
| 연맹당 사용 가능 횟수 | **총 2회** | 전장 내 총량 제한 |
| 사용 쿨타임 | **12시간** | 1회 사용 후 다음 사용까지 |
| 대상 도시 | 7도시만 해당 | 6도시·8도시 미적용 |

**코인추격 동작 방식**
```
초기 상태: 7도시 보유 연맹 = 450,000점
1회 추격 성공: 보유 연맹 -90,000점 / 공격 연맹 +90,000점
→ 7도시 4개 × 450,000 = 총 1,800,000 프리즘코인 풀

연맹 A의 코인추격 타임라인 예시:
  T+0h:  1회 사용 → 90,000점 획득
  T+12h: 쿨타임 종료
  T+12h: 2회 사용 → 90,000점 추가 획득
  → 최대 180,000점 획득 가능 (2회 × 90,000)
```

**프리즘코인 귀속 규칙**

```
코인추격 성공 시:
  case 1: 7도시 미보유 → 연맹 포인트로만 귀속 (도시 풀 외)
  case 2: 7도시 1개 보유 → 해당 7도시 프리즘코인 풀에 합산
  case 3: 7도시 2개 이상 보유 → 가장 최근 점령한 7도시 풀에 합산

도시 점령 이전 시:
  프리즘코인을 보유한 7도시가 다른 연맹에 점령되면
  → 해당 프리즘코인은 점령한 연맹 소유로 이전
  → 재점령 시 재이전 (도시와 함께 이동)

영구귀속:
  코인추격 가능 기간 종료 후
  → 해당 시점 보유 연맹에 영구귀속
```

**점수 종류 구분**

| 종류 | 획득 방법 | 합산 기준 |
|------|-----------|-----------|
| 도시 점령 점수 | 도시 점령 유지 | 점령 중인 도시 점수 합산 |
| 프리즘코인 (코인추격) | 코인추격 전쟁 선포 성공 | 도시 보유 시 도시 풀, 미보유 시 연맹 포인트 |

### 시뮬레이터 출력

- 각 연맹별 현재 점령 도시 목록 및 점수
- 연맹별 프리즘코인 현재 보유량 입력 → 총점 계산
- 동맹 관계 기준 진영별 총점 비교 (도시점수 + 프리즘코인)
- "이 경로로 가면 몇 점 획득 가능" 예측

```
  → 총 2회 × 90,000 = 180,000점 획득
```

### 시뮬레이터 반영 사항

```
현재 일차(1/2/3일차) 선택 → 개방된 도시만 활성화
UTC 기준 현재 시각 표시
코인추격 가능 여부 표시 (토요일 이후)
코인추격 쿨타임 잔여 시간 표시 (12시간 기준)
```

## 5-E. 동맹 관계 시스템

### 개요

8개 연맹 간의 동맹(ally) / 적대(enemy) 관계를 설정하고, 진영별 합산 점수와 전략을 시뮬레이션한다.

### 관계 종류

| 관계 | 표시 | 의미 |
|------|------|------|
| 동맹 | 🟢 | 점수 합산, 타워 라인 통과 허용 |
| 중립 | ⚪ | 기본값, 관계 없음 |
| 적대 | 🔴 | 충돌 계산 대상, 타워 라인 차단 |

### 관계 매트릭스

8×8 연맹 관계 표를 UI에서 설정 (대각선 제외 28쌍).

```json
"relations": {
  "A-B": "ally",
  "A-C": "enemy",
  "B-C": "neutral",
  ...
}
```

### 기능 요구사항

```
동맹 연맹끼리 색상 그룹으로 표시 (같은 계열 색)
동맹 진영 총점 자동 합산
적대 연맹과의 예상 충돌 좌표 계산
동맹 연맹의 타워 라인 통과 여부 표시 (후순위)
```
